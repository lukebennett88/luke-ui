`Emoji` renders an emoji with a reliable accessible name. It accepts the same typography props as
`Text`.

## Typography

`Emoji` accepts every `Text` prop except `children` and `elementType`, including `size`, semantic
`color`, and `fontWeight`.

## Accessibility

Both `emoji` and `label` are required. Screen readers announce the label instead of the raw emoji,
which is not announced consistently across platforms.

Write a label that describes the emoji's meaning in context, such as "Celebration". Do not repeat
text that is already in the surrounding sentence, because that creates duplicate screen reader
output.

## Related components

Use [Text](/components/typography/text) when the content is text, and
[Icon](/components/visuals/icon) for interface symbols from Luke UI's icon set.
