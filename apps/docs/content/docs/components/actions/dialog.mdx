`Dialog` presents content in a focused overlay, blocking interaction with the page behind it. Use it
for confirmations, forms, and other tasks that benefit from a dedicated layer.

```tsx
import { Dialog, DialogTrigger } from '@luke-ui/react/dialog';
import { Button } from '@luke-ui/react/button';

<DialogTrigger>
	<Button>Open Dialog</Button>
	<Dialog>Dialog content</Dialog>
</DialogTrigger>;
```

## Anatomy

A dialog is composed of several subcomponents that build on
[React Aria Components](https://react-spectrum.adobe.com/react-aria-components.html).

## Close behaviour

By default clicking outside the dialog or pressing <kbd>Escape</kbd> dismisses it. Set
`isDismissable={false}` on `Dialog` to require an explicit action.

The `Dialog` component renders a close button in the top-right corner by default. Set
`showCloseButton={false}` to remove it.

A text close button can be placed in the footer with `showCloseButton` on `DialogFooter`.

```tsx
<DialogTrigger>
	<Button>Open Dialog</Button>
	<Dialog>
		<DialogTitle>Confirm</DialogTitle>
		<DialogFooter showCloseButton>
			<Button tone="accent">Confirm</Button>
		</DialogFooter>
	</Dialog>
</DialogTrigger>
```

## Accessibility

The dialog manages focus automatically: focus moves inside when opened and returns to the trigger
when closed. The overlay receives `role="dialog"` and is labelled by the `DialogTitle` component.
