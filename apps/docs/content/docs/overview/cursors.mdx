---
title: Cursors
description: Understand browser-default cursors and Luke UI's interaction conventions.
---

Cursors reinforce what an element can do. Luke UI keeps browser defaults where they already describe
the interaction, then adds a small set of explicit cursors for library controls.

## Browser defaults

Text inputs and text areas use the browser's `text` cursor because people can place a caret and edit
their value. Static content keeps the browser default. Luke UI sets a cursor only when it describes
the interaction.

Use a component's disabled or pending API instead of setting its cursor yourself. The API also
communicates the state to assistive technology and blocks the interaction itself.

## Resize affordances

Luke UI does not expose a resizable component or cursor token. The docs playground has a resizable
editor and preview separator: it uses `row-resize` when the panes are stacked on narrow screens and
`col-resize` when they sit side by side from the medium breakpoint. The separator has a larger hit
area and a visible handle, so the cursor confirms an affordance that is already visible.

When you build a resize control, match the cursor to the direction of movement. Do not use a resize
cursor on a static divider.
