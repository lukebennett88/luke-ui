Author a custom theme when the bundled identities do not suit the product. `buildTheme` turns a
typed foundation into static CSS for one identity with light and dark modes. It does not need a
browser or a Luke UI release.

## Define the foundation

A `ThemeFoundation` includes:

- a kebab-case `name`
- independent `light` and `dark` mode foundations
- `color.neutral` and `color.accent` source colours for each mode
- `actionControlFinish` backgrounds for recessed, resting, and raised controls
- `depth` shadows for recessed, resting, raised, floating, and overlay surfaces

Optional source colours cover info, success, warning, danger, and focus. Luke UI supplies accessible
mode-specific defaults when they are omitted. Optional shared values choose a supported
Capsize-compatible font, its weight roles, and the detail, control, surface, and overlay radii.

## Contrast validation

`buildTheme` calculates the generated colour values in OKLCH, maps them to the sRGB gamut, and
checks WCAG 2.2 AA contrast. Text pairs must reach 4.5:1. Non-text UI pairs must reach 3:1.

When validation fails, `buildTheme` throws `ThemeContrastError`. Its `failures` array identifies
each colour mode and token pair, so the stylesheet is not written with an invalid combination.
